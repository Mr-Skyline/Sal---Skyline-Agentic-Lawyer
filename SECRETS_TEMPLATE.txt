Dispute Agent Elite — where secrets live
======================================================================

Canonical project folder (this machine): C:\Users\travi\Projects\AI Lawyer Build
(Keep credentials.json, token.pickle, and .env only in this folder — no second project copy.)

1) API keys → .env (created automatically from the app sidebar, or paste manually)

   XAI_API_KEY=your_xai_key
   ZHIPU_API_KEY=your_zhipu_key
   GROK_MODEL=optional_override_if_xai_changes_model_ids

   GLMOCR_API_KEY is also accepted by glmocr instead of ZHIPU_API_KEY.

2) Gmail OAuth → credentials.json

   Google Cloud Console → APIs & Services → Credentials → Create OAuth client ID
   → Application type: Desktop app → Download JSON → save as credentials.json
   in the same folder as main.py.

3) After first Gmail sign-in, token.pickle is written automatically (do not share).

These files are listed in .gitignore. Do not commit them.

4) Sanity check (optional)

   py verify_setup.py
   py verify_setup.py --strict

5) Optional — Sal review Markdown folder

   SKYLINE_REVIEW_DIR=C:\Users\travi\Projects\AI Lawyer Build\skyline_review
   (Default is .\skyline_review under the project root if unset. Project root for this build is only the Projects path above.)

6) Optional — continuous Gmail archive (sync_worker.py)

   AGENT_GMAIL_ADDRESS=agent@yourdomain.com
   CORRESPONDENCE_ARCHIVE_DIR=\\server\share\LegalAgent\archive

7) Optional — Supabase thread index (after running supabase_schema.sql in the project)

   SUPABASE_URL=https://xxxx.supabase.co
   SUPABASE_SERVICE_ROLE_KEY=...   (server-side only; never in a frontend)

If you paste keys to an AI assistant in chat, assume they may be retained in
that chat’s history — prefer the in-app “Store secrets” sidebar instead.
